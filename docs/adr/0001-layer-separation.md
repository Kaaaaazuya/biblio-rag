# 0001. 層分離と中間成果物（normalized / chunks を正本）

- 状態: Accepted
- 日付: 2026-06-16
- 関連: [design.md](../design.md) §2

## コンテキスト

取り込みは ①抽出 → ②チャンク → ③埋め込み の3段。チューニングの大半は②③（チャンクサイズ・
埋め込みモデル・DB）で発生する。最も重い①を毎回やり直すのは無駄が大きい。

## 決定

各層が中間成果物をファイルとして残し、それを「正本」とする。

- ① の出力 `normalized/*.md` を正本にする → ② のチューニングは ① を再実行せず済む。
- ② の出力 `chunks/*.jsonl` を正本にする → ③ のチューニング（埋め込み・DB）は ①② を再実行せず済む。

「正本」は再チューニング用の元データの意味で、保管先はローカルディスク（開発）/ S3（本番）。
git では管理しない（本文＝著作権物のため。[README](../../README.md) のデータ取り扱いルール参照）。

## 結果

- 良い点: 変更の影響範囲が層で閉じ、再実行コストが最小。本番移行も `chunks/*.jsonl` の再埋め込みで済む。
- 悪い点: 中間ファイルのストレージと世代管理が必要。
- 制約: ② は `normalized/*.md` のみを入力とし PDF を再オープンしない（層境界を守る）。
  そのため page 等 PDF 固有のメタデータが必要なら ① が出力に含める必要がある。
